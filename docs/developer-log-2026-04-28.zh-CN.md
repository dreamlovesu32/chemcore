# Chemcore 开发者日志 - 2026-04-28

作者：张家骏

时间范围：2026-04-28 00:00 至 2026-04-28 23:59，Asia/Shanghai

## 总结

今天这轮工作的主线，不是继续在 viewer 里补局部偏差，而是把“cm 几何是唯一真相”这件事再往前推了一大步。最核心的变化有四类：

- 文本编辑态的几何、锚点和显示层继续从浏览器逻辑往内核收；
- `engine.rs` 里过重的文本编辑、删除、选择逻辑开始拆分成独立模块；
- 选择与删除两条交互链正式接上，且分子内多选规则按化学编辑器预期收敛；
- 键选中框不再只包主键中心线，而是按实际渲染出来的整根键轮廓计算，双键、三键、粗键和楔形键都能正确包住。

这轮最大的收益，是把几类过去最容易“看起来差一点”的问题，从浏览器测量和前台补偿，逐步收敛到内核几何和渲染共用结果上。

## 文本编辑态继续内核化

前面已经把文本编辑从 `contenteditable` 语义层往外拆了一轮，今天继续做的，是把编辑态真正容易漂移的那部分几何继续收口。

### 编辑态显示与输入解耦

编辑态现在继续沿用“隐藏输入层 + 可见 SVG 显示层”的结构：

- `textarea` 只承接键盘和输入法；
- 可见文字、黑色光标、蓝色选区、黑框都走自绘显示；
- viewer 不再靠 DOM 的真实排版结果反推文本几何。

这一步的目的很直接：浏览器可以负责输入，但不再负责文本几何真相。

### 文本框锚点修正

文本框进入编辑态时，锚点语义做了进一步统一。现在编辑框更接近“I 光标左上角为起点”的模型，而不是落在字形中心附近再做补偿。

这件事虽然看起来像交互细节，但本质上是在消掉“点击点、编辑框起点、提交后文字位置”三者不一致的问题。

### 历史 DOM 尺寸字段退出边界

`measuredSize` 这类历史字段从提交链路里被继续边缘化，viewer 不再把 DOM 测出来的尺寸当成内核几何输入。文本和标签的几何，逐步只认内核 layout 和明确的 cm 边界值。

## Rust 引擎拆分

这轮还顺手处理了一个结构性问题：`crates/chemcore-engine/src/engine.rs` 已经承载了太多不该继续堆在一起的逻辑。

今天拆出去的主要模块有：

- `crates/chemcore-engine/src/engine/text_edit.rs`
- `crates/chemcore-engine/src/engine/delete.rs`
- `crates/chemcore-engine/src/engine/select.rs`

这样做的收益有两点：

- 文本编辑、删除、选择三条高复杂度链路不再互相缠在一个大文件里；
- 后面继续做交互修正时，定位范围明显缩小，不需要再在 `engine.rs` 主文件里来回翻。

## 选择与删除交互收口

今天选择和删除这两块，已经从“能工作”继续往“规则统一”推进了一轮。

### 删除模式

删除模式现在按专用工具工作，而不是只依赖顶部命令：

- 切到删除工具后，光标进入删除态；
- 点击键、端点、标签、文本框时分别走对应的化学删除语义；
- 普通模式下按 `Delete`，则保持“删除当前聚焦对象”的逻辑。

### 选择模式

选择模式现在支持：

- 点选；
- 矩形框选；
- 套索选；
- `Shift` 追加多选。

更重要的是，分子内部的显示规则按一套统一逻辑收敛了：

- 分子内只选中一个对象时，显示这个对象自己的框和中心蓝点；
- 分子内选中多个对象时，只显示包裹这些对象的最小外框，内部对象只保留中心蓝点；
- 不同分子分别计算各自的最小框。

标签、原子、键都已经并入这套规则，不再各走一套显示路径。

## 键选中框改为真实轮廓包围盒

今天最后一块用户可见问题，是键选中框仍然按主键中心线加固定宽度算 bbox，所以双键、三键选中时明显偏小。

这次没有继续在 `select.rs` 里补特判，而是直接改成复用渲染结果：

- 在 `render.rs` 里新增 `fragment_bond_visual_bounds(...)`；
- 它直接复用现有 bond 渲染逻辑，为单根键生成实际 `DocumentBond` primitives；
- 然后对这根键渲染出的 line / polygon / polyline 求真实包围盒；
- `select.rs` 只消费这个包围盒，不再自己猜双键、三键、粗键的外轮廓。

这样做之后：

- 双键选中框会包住两根线；
- 三键选中框会包住三根线；
- 粗键、楔形键等宽轮廓键也会跟着正确扩展；
- 后续如果 bond 渲染参数继续调整，选中框会自然跟着走，不需要再手动同步一套偏移常量。

## 前端边界继续收紧

viewer 这边这轮不是加新算法，而是继续减前端几何责任：

- `viewer/app.js` 里的文本编辑控制流继续瘦身；
- `viewer/text_editor_render.js` 里旧的几何壳层继续被清掉；
- wasm 导出和前端绑定一起更新，确保 viewer 走的是新的内核接口和新生成的包。

这意味着 viewer 越来越接近“输入事件 + 世界坐标转换 + primitive 渲染”的角色，而不是继续承担一套隐藏的文本/标签几何系统。

## 验证

今天这轮主要跑过的验证有：

```bash
cargo test -p chemcore-engine --test bond_tool --test text_tool
npm run build:engine-wasm
```

其中重点覆盖了：

- 文本编辑工具的回归测试；
- 标签与 attached label 的交互测试；
- 删除与选择链路；
- 单键、双键、三键的选中框回归；
- wasm 包重新生成后的 viewer 绑定一致性。

## 风险和下一步

今天虽然已经把文本编辑几何和键选中框都往正确方向推进了，但还有两件事后面仍值得继续做：

- 文本编辑虽然已经不再主要依赖 DOM 几何，但 caret、selection、IME 这条链还可以继续往内核结果驱动收紧；
- bond 选中框现在已经走真实渲染包围盒，后续如果还要做更复杂的多段图元或特殊箭头/图形选中，最好继续沿用“渲染即真相”的方式，而不是回到手写近似框。

今天这轮工作的结论很简单：  
几何真相继续从浏览器和前台补偿手里收回来了，内核和渲染开始更像唯一来源，这对后续多端同步和持续维护都是必要的。
